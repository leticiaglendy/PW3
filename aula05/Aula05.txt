GitHub Branch/Main

1. usuário do amigo no GitHub
2. convidar para o repositório
3. não é possível subir a pasta vazia no repositório
4. clonar o repositório que foi convidado
5. Projeto <Main> é onde tudo que já está pronto é enviado(principal), se cria uma cópia que representa(homologação) e para o ambiente de desenvolvimento existe uma outra cópia onde pode ser criada as <branch>
6. comando: 
	git checkout -b front/muda-cor-botão
	git checkout -b teste/leticia-glendy
7.Alteração no arquivo README.md
8. comando na pasta/local desejada:
	git status
	add add .
9. como fazer o comando numa branch:
	git push --set-upstream origin teste/leticia-glendy
10. ver/puxar as alterações criadas no projeto/repositorio:
	git pull
11. voltar de uma Branch ao Main:
	git checkout main
12. Voltar para a branch desejada:
	git checkout teste/leticia-glendy
